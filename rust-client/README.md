# Net Port Proxy - Rust Implementation

Это реализация сетевого прокси-сервера на языке Rust, портированная с оригинального C-проекта. Сервер выполняет форвардинг трафика между input и output соединениями с поддержкой множественных подключений.

## Особенности

- **Многопоточность**: Поддержка множественных одновременных соединений
- **Асинхронность**: Использование async/await для эффективной обработки
- **Безопасность**: Гарантии безопасности памяти Rust
- **Graceful Shutdown**: Корректное завершение работы с ожиданием завершения соединений
- **Логирование**: Комплексная система логирования с ротацией файлов
- **Конфигурация**: Гибкая настройка через аргументы командной строки

## Архитектура

Проект состоит из следующих модулей:

- **`config.rs`** - обработка аргументов командной строки и конфигурация
- **`connection.rs`** - управление соединениями и их состоянием
- **`input_thread.rs`** - обработка входящих соединений
- **`output_thread.rs`** - обработка исходящих соединений
- **`logger.rs`** - система логирования
- **`shutdown.rs`** - механизм graceful shutdown
- **`error.rs`** - обработка ошибок

## Установка и запуск

### Требования

- Rust 1.60+ и Cargo
- Tokio runtime

### Сборка

```bash
cd rust-client
cargo build --release
```

### Запуск

```bash
# Базовая конфигурация
cargo run --release

# Специфичная конфигурация
cargo run --release -- \
    --host_in 82.146.44.140 \
    -p_in 6000 \
    --host_out 127.0.0.1 \
    -p_out 22 \
    --connections 5 \
    --timeout 60
```

## Параметры командной строки

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `--host_in` | Адрес input сервера | `82.146.44.140` |
| `-p_in` | Порт input сервера | `6000` |
| `--host_out` | Адрес output сервера | `127.0.0.1` |
| `-p_out` | Порт output сервера | `22` |
| `-c, --connections` | Количество соединений | `1` |
| `-t, --timeout` | Таймаут бездействия (секунды) | `1200` |
| `--help` | Показать справку | - |

## Примеры использования

```bash
# Проксирование SSH соединения
./net-port-proxy --host_in 82.146.44.140 -p_in 6000 --host_out 127.0.0.1 -p_out 22

# Множественные соединения с таймаутом
./net-port-proxy --connections 10 --timeout 300

# Кастомная конфигурация
./net-port-proxy \
    --host_in 192.168.1.100 \
    -p_in 8080 \
    --host_out 10.0.0.50 \
    -p_out 443 \
    --connections 3 \
    --timeout 120
```

## Логирование

Логи сохраняются в файл `logs/net_port_proxy.log` с ротацией. Уровни логирования соответствуют стандарту syslog:

- `EMERG` (0) - Система неработоспособна
- `ALERT` (1) - Требуются немедленные действия
- `CRIT` (2) - Критические условия
- `ERR` (3) - Ошибки
- `WARNING` (4) - Предупреждения
- `NOTICE` (5) - Важные нормальные условия
- `INFO` (6) - Информационные сообщения
- `DEBUG` (7) - Отладочные сообщения

## Graceful Shutdown

Приложение поддерживает корректное завершение работы по сигналам:
- `Ctrl+C` (SIGINT)
- `SIGTERM`

При получении сигнала приложение:
1. Останавливает прием новых соединений
2. Завершает текущие передачи данных
3. Закрывает все активные соединения
4. Корректно завершает работу

## Отличия от C-версии

1. **Безопасность памяти**: Rust гарантирует отсутствие ошибок памяти
2. **Асинхронность**: Использование async/await вместо блокирующих вызовов
3. **Error handling**: Комплексная система обработки ошибок
4. **Типизация**: Строгая типизация конфигурации и данных
5. **Модульность**: Четкое разделение ответственности между модулями

## Производительность

Благодаря асинхронной архитектуре и эффективному использованию ресурсов, Rust-версия показывает:
- Меньшее потребление памяти
- Лучшую масштабируемость
- Более предсказуемое поведение под нагрузкой

## Разработка

### Структура проекта

```
rust-client/
├── src/
│   ├── main.rs          # Точка входа
│   ├── config.rs       # Конфигурация
```

# Техническая спецификация: Динамическое перенаправление портов с идентификацией клиентов

## Краткое описание

Расширение существующей системы net_port для поддержки динамического назначения портов и идентификации клиентов IoT устройств с двусторонней коммуникацией. Система позволяет устройствам подключаться к центральному серверу, получать уникальный внешний порт, через который внешние клиенты могут подключаться к устройству.

## Основные возможности

1. **Динамическое выделение портов**: Автоматическое назначение уникальных портов для каждого устройства
2. **Идентификация клиентов**: Уникальный идентификатор устройства + токен аутентификации
3. **Двусторонняя коммуникация**: Прозрачное туннелирование TCP/UDP трафика
4. **Масштабируемость**: Поддержка тысяч одновременных подключений
5. **Безопасность**: End-to-end шифрование, аутентификация, авторизация
6. **Мониторинг**: Real-time статистика, алертинг, логирование
7. **Управление**: Веб-интерфейс для администраторов

## Архитектурные решения

### 1. Многоуровневая архитектура
```
Уровень 1: Сетевой (порты 8443, 10000-60000)
Уровень 2: Транспортный (SSL/TLS)
Уровень 3: Прикладной (протокол регистрации/аутентификации)
Уровень 4: Данные (туннелирование трафика)
```

### 2. Компонентная архитектура
- **Control Server** (порт 8443): Управление подключениями, аутентификация
- **Proxy Server** (порты 10000-60000): Туннелирование трафика
- **Database**: Хранение устройств, сессий, статистики
- **Web Interface**: Управление через браузер
- **IoT Client**: Модифицированный net_port клиент

### 3. Протокол взаимодействия
```
1. Устройство → Control Server: Регистрация + аутентификация
2. Control Server → Устройство: Назначенный порт + session token
3. Устройство → Control Server: Периодический heartbeat
4. Внешний клиент → Proxy Server: Подключение к назначенному порту
5. Proxy Server → Устройство: Туннелирование соединения
```

## Технические детали

### Диапазон портов
- **Control порт**: 8443 (SSL/TLS обязателен)
- **Data порты**: 10000-60000 (динамическое выделение)
- **Максимум устройств**: 50000 одновременных подключений

## Схема базы данных

### Таблица devices
```sql
CREATE TABLE devices (
    id UUID PRIMARY KEY,
    device_id VARCHAR(64) UNIQUE,
    name VARCHAR(255),
    auth_token_hash VARCHAR(255),
    status VARCHAR(20),
    assigned_port INTEGER,
    internal_address VARCHAR(45),
    internal_port INTEGER,
    capabilities JSONB,
    metadata JSONB,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    last_heartbeat TIMESTAMP
);
```

### Таблица sessions
```sql
CREATE TABLE sessions (
    id UUID PRIMARY KEY,
    device_id UUID REFERENCES devices(id),
    session_token VARCHAR(255),
    client_ip VARCHAR(45),
    started_at TIMESTAMP,
    last_activity TIMESTAMP,
    expires_at TIMESTAMP,
    bytes_sent BIGINT,
    bytes_received BIGINT
);
```

### Таблица port_allocations
```sql
CREATE TABLE port_allocations (
    port INTEGER PRIMARY KEY,
    device_id UUID REFERENCES devices(id),
    allocated_at TIMESTAMP,
    expires_at TIMESTAMP
);
```

## Безопасность

### 1. Аутентификация
- **Устройства**: Pre-shared token + SSL client certificate (опционально)
- **Пользователи**: JWT tokens через веб-интерфейс
- **Сессии**: Time-limited session tokens с автоматическим обновлением

### 2. Шифрование
- **Control канал**: TLS 1.2+ с сильными шифрами
- **Data канал**: TLS или raw TCP (на выбор)
- **Сертификаты**: Let's Encrypt или корпоративный CA

### 3. Защита от атак
- **Rate limiting**: 10 запросов/секунду на устройство
- **IP whitelisting**: Ограничение по IP адресам
- **Port scanning protection**: Обнаружение сканирования портов
- **DDoS protection**: Интеграция с Cloudflare/Akamai

### 4. Аудит и логирование
- Все операции аутентификации
- Выделение/освобождение портов
- Статистика трафика
- Ошибки и исключения

## Производительность

### Целевые метрики
- **Задержка**: < 100ms для 95% запросов
- **Пропускная способность**: 1Gbps на сервер
- **Количество устройств**: 1000+ одновременных подключений
- **Память**: < 1MB на активное соединение
- **CPU**: < 20% при полной нагрузке

### Оптимизации
- **Connection pooling**: Для БД соединений
- **In-memory cache**: Redis для частых запросов
- **Async I/O**: Неблокирующие операции
- **Batch processing**: Для статистики и логирования

## Масштабирование

### Вертикальное масштабирование
- Увеличение диапазона портов
- Добавление CPU/RAM
- Оптимизация конфигурации БД

### Горизонтальное масштабирование
- **Балансировщик нагрузки**: Распределение устройств между серверами
- **Шардирование**: По диапазонам портов или device_id
- **Репликация БД**: Master-slave для чтения/записи

### Кластеризация
- **Active-Active**: Несколько серверов с общим состоянием
- **Shared storage**: Для сессий и маппинга портов
- **Service discovery**: Автоматическое обнаружение серверов

## Мониторинг и алертинг

### Ключевые метрики
```yaml
system:
  cpu_usage: < 80%
  memory_usage: < 85%
  disk_usage: < 90%
  network_throughput: мониторинг

application:
  active_devices: количество
  active_sessions: количество
  ports_used: процент использования
  authentication_rate: успешные/неуспешные
  heartbeat_missed: количество пропущенных

business:
  traffic_per_device: байты/секунду
  connection_duration: среднее время
  error_rate: процент ошибок
  uptime: доступность системы
```

### Алёрты
1. **Критические**: Отказ сервера, исчерпание портов, атаки
2. **Важные**: Высокая загрузка, много ошибок аутентификации
3. **Информационные**: Новые устройства, необычные паттерны трафика

## Развертывание

### Требования к окружению
- **ОС**: Linux (Ubuntu 20.04+, CentOS 8+)
- **БД**: PostgreSQL 12+
- **Кэш**: Redis 6+
- **Память**: 8GB+ RAM
- **Диск**: 50GB+ SSD
- **Сеть**: 1Gbps+ bandwidth

### Процесс развертывания
```bash
# 1. Установка зависимостей
apt-get install postgresql redis openssl

# 2. Настройка БД
psql -c "CREATE DATABASE net_port_devices;"
psql -d net_port_devices -f schema.sql

# 3. Сборка сервера
cd server
mkdir build && cd build
cmake .. && make

# 4. Конфигурация
cp config.example.json config.json
# редактирование config.json

# 5. Запуск
./module_net_port_server --config config.json
```

### Docker развертывание
```dockerfile
FROM ubuntu:20.04
# ... сборка приложения
```

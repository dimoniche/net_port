# Net Port Installation Guide

This guide explains how to install the Net Port system (web interface, server, and database) without Docker using the provided installation script.

## Prerequisites

- Linux system (Debian/Ubuntu or RHEL/CentOS recommended)
- Root access or sudo privileges
- At least 2GB of free disk space
- Internet connection for downloading dependencies

## Quick Installation

1. Download or clone the Net Port repository
2. Run the installation script as root with required credentials:

**Using command-line arguments:**
```bash
sudo ./install.sh --db-user netport --db-password "secure_db_password" --admin-password "admin_web_password"
```

**Using environment variables:**
```bash
sudo DB_USER=netport DB_PASSWORD=secure_db_password APP_PASSWORD=admin_web_password ./install.sh
```

**Interactive mode (script will prompt for missing credentials):**
```bash
sudo ./install.sh
```

### Available Options:
- `--db-user USER`: PostgreSQL database user (required)
- `--db-password PASSWORD`: PostgreSQL database password (required)
- `--db-name NAME`: Database name (default: net_port)
- `--db-host HOST`: Database host (default: localhost)
- `--db-port PORT`: Database port (default: 5432)
- `--admin-user USER`: Admin user for web interface (default: admin)
- `--admin-password PASS`: Admin password for web interface (required)
- `--no-prompt`: Do not prompt for missing credentials
- `--help`: Display help message

The script will:
- Install all required dependencies (build tools, PostgreSQL, Node.js, nginx)
- Build the C server and client from source
- Set up PostgreSQL database with initialization using provided credentials
- Install and configure Node.js backend
- Build the React frontend
- Configure nginx as a web server
- Create systemd services for automatic startup
- Generate SSL certificates (self-signed)
- Create admin user with provided credentials

## Manual Installation Steps

If you prefer to install manually or troubleshoot, here are the main steps:

### 1. Install System Dependencies

**Debian/Ubuntu:**
```bash
sudo apt-get update
sudo apt-get install -y build-essential cmake git postgresql postgresql-contrib \
    nginx nodejs npm openssl libssl-dev pkg-config curl wget python3 python3-pip
```

**RHEL/CentOS:**
```bash
sudo yum install -y gcc gcc-c++ make cmake git postgresql-server postgresql-contrib \
    nginx nodejs npm openssl openssl-devel pkgconfig curl wget python3 python3-pip
sudo postgresql-setup --initdb
```

### 2. Build C Applications

```bash
mkdir build
cd build
cmake ..
make -j$(nproc)
```

### 3. Set Up PostgreSQL

Replace `DB_USER`, `DB_PASSWORD`, and `DB_NAME` with your chosen credentials:

```bash
DB_USER="netport"
DB_PASSWORD="secure_password"
DB_NAME="net_port"

sudo systemctl start postgresql
sudo systemctl enable postgresql
sudo -u postgres psql -c "CREATE USER $DB_USER WITH PASSWORD '$DB_PASSWORD';"
sudo -u postgres psql -c "CREATE DATABASE $DB_NAME;"
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE $DB_NAME TO $DB_USER;"
sudo -u postgres psql -d $DB_NAME -f init_db.sql
```

### 4. Install Node.js Backend

```bash
cd web/backend_net_port
npm install

# Create .env file with database credentials
cat > .env << EOF
DB_USER=$DB_USER
DB_PASSWORD=$DB_PASSWORD
DB_HOST=localhost
DB_PORT=5432
NODE_ENV=production
EOF
```

Create `.env` file:
```bash
cat > .env << EOF
DB_USER=admin
DB_PASSWORD=password
DB_HOST=localhost
DB_PORT=5432
NODE_ENV=production
EOF
```

### 5. Build React Frontend

```bash
cd ../frontend_net_port
npm install
npm run build
```

### 6. Configure Nginx

Copy the nginx configuration from `nginx.conf` or use the one generated by the install script.

### 7. Create Systemd Services

See the `install.sh` script for example service files.

## Post-Installation

### Accessing the Web Interface

After installation, access the web interface at:
- `http://your-server-ip/`

Default credentials (if using add_test_user.js):
- Username: `admin`
- Password: `password`

### Managing Services

```bash
# Start all services
sudo systemctl start net-port-backend net-port-server

# Stop all services
sudo systemctl stop net-port-backend net-port-server

# Check status
sudo systemctl status net-port-backend net-port-server

# View logs
sudo journalctl -u net-port-backend -u net-port-server -f
```

### Using the Management Script

The installation creates a management script at `/opt/net_port/manage.sh`:

```bash
sudo /opt/net_port/manage.sh start    # Start services
sudo /opt/net_port/manage.sh stop     # Stop services
sudo /opt/net_port/manage.sh restart  # Restart services
sudo /opt/net_port/manage.sh status   # Check status
sudo /opt/net_port/manage.sh logs     # View logs
```

## Uninstallation

To completely remove Net Port:

```bash
sudo /opt/net_port/uninstall.sh
```

Or manually:
1. Stop and disable services
2. Remove systemd service files
3. Remove nginx configuration
4. Delete installation directory (`/opt/net_port`)
5. Optionally remove PostgreSQL database and user

## Troubleshooting

### Common Issues

1. **PostgreSQL connection errors**
   - Check if PostgreSQL is running: `sudo systemctl status postgresql`
   - Verify credentials in `.env` file
   - Check PostgreSQL logs: `sudo journalctl -u postgresql`

2. **Node.js backend fails to start**
   - Check Node.js version: `node --version` (should be 14+)
   - Verify dependencies: `npm install` in backend directory
   - Check logs: `sudo journalctl -u net-port-backend`

3. **Nginx configuration errors**
   - Test configuration: `sudo nginx -t`
   - Check nginx logs: `sudo tail -f /var/log/nginx/error.log`

4. **Port conflicts**
   - Default ports: 80 (nginx), 8080 (backend), 5432 (PostgreSQL)
   - Check for conflicts: `sudo netstat -tulpn | grep -E ':80|:8080|:5432'`

### Log Files

- Installation log: `/var/log/net_port_install.log`
- Backend logs: `journalctl -u net-port-backend`
- Server logs: `journalctl -u net-port-server`
- Nginx logs: `/var/log/nginx/access.log` and `/var/log/nginx/error.log`
- PostgreSQL logs: `/var/log/postgresql/postgresql-*.log` or `/var/lib/pgsql/data/log/`

## Security Considerations

1. **Change default passwords**
   - Database password in `/opt/net_port/source/web/backend_net_port/.env`
   - Admin user password in the web interface

2. **Firewall configuration**
   - Allow ports 80 (HTTP) and optionally 443 (HTTPS)
   - Restrict access to port 5432 (PostgreSQL) to localhost only

3. **SSL certificates**
   - Replace self-signed certificates with valid ones for production
   - Configure nginx to use HTTPS

4. **Regular updates**
   - Keep system packages updated
   - Monitor for security updates for Node.js and PostgreSQL
